Kingfisher Help Center

Trade with structure, not guesswork.

Kingfisher is a positional MT5 trading bot built around cycle management, spacing rules, basket logic, and account-level safety controls. This page replaces the old Notion guide with a version that is easier to host as a static help page.

What Kingfisher Does
  • Opens an initial market position when the strategy confirms a valid direction.
  • Places additional pending orders at configured spacing levels.
  • Builds and manages baskets instead of depending on a single fast scalp.
  • Supports lot progression, pending ladders, and cycle-level profit logic.
  • Applies account safety rules like target achievement, loss control, and aggressive mode.
Current Model

Older internal names such as ema_scalper_settings still exist for compatibility, but the live behavior is now driven by a positional cycle engine. Kingfisher is designed to manage exposure over time, not just jump in and out of the market in seconds.

Overview

How the current positional model works

1. Strategy decides direction

The active strategy produces a BUY or SELL bias.

2. Kingfisher opens a position

The bot places the first market order for the cycle.

3. Pending ladder can expand

Loss-side pending orders may be added using the configured spacing rules.

4. Profit or safety logic exits

The bot can close single trades, close the basket, stop, or shut down.

Kingfisher is positional because it manages a full trading cycle over minutes or hours, not just a single rapid scalp.
Config File

preferences.json

This file controls app behavior, hotkeys, remembered user choices, notifications, updates, and account-level safety rules.

hotkeys

Controls quick actions for show or hide app, show or hide terminal, start bot, and stop bot.

notifications

Defines how app alerts appear, including internet-connectivity warnings.

last_symbol_selection

Stores the last selected symbol, whether it should load on startup, and the broker suffix behavior.

target_achievement

Stops or closes trading once a configured gain target has been reached.

loss_control

Main drawdown guard for closing or stopping the bot when loss thresholds are hit.

aggressive_mode

Optional compounding mode that increases lot behavior after profit conditions are met.

updates

Controls update checks, automatic downloads, installation behavior, and retained versions.

Config File

symbol_config.json

Each top-level key in this file is a trading symbol such as BTCUSDm or XAUUSDm. Every symbol can have its own signal inputs, lot rules, spacing, and cycle behavior.

ema_scalper_settings

Legacy name for the section that still defines signal quality inputs such as EMA length, pip size, and trading zone values.

execution

Contains order execution rules such as slippage tolerance.

lot_management

Defines base lot, lot precision, min and max lot, bump settings, and starting size.

magic

Specifies the MT5 magic number used to isolate Kingfisher trades from manual or external ones.

timeframe

Sets the chart timeframe used by the strategy for that symbol.

trade_behavior

The most important symbol-specific section. It controls cycle limits, spacing, lot progression, hedging, and safety rules.

Deep Dive

trade_behavior fields that matter most

Core capacity

max_allowed_batches, max_allowed_trades, max_allowed_pos, and min_open_interval_s control how quickly and how deeply the cycle can expand.

Pending-order ladder

pending_orders_limit, pending_orders_streak, and rung_count_added_per_fill define the shape of the pending ladder.

Spacing

spacing_usd takes priority when present. Otherwise the bot uses spacing_pips together with pip_size. Progressive spacing can be enabled with progressive_gap.

Lot progression

starting_lot, additive_factor, and progressive position lot settings determine how exposure grows as the cycle expands.

Profit and close behavior

single_trade_profit, counter-position close settings, and flash-image options define how the bot takes profit and reports closures.

Advanced safety

max_cycle_drawdown_usd and cycle_drawdown_csv_enabled support protective cycle monitoring and post-run analysis.

Change advanced acceleration, hedge, and drawdown settings carefully. These keys directly affect how the cycle behaves under pressure.
